Let's get something playing. By the end of this page you will have lcluster running, a node up, and your bot connected. Five minutes. Let's go.

## Step 3 â€” Open the Dashboard

Time to see the fruits of your labor. Drop into the dashboard.

```bash

```

<Note>

</Note>

## Step 4 â€” Connect Your Bot

Here is the magic trick. You don't configure your bot to talk to the node you just created. You configure it to talk to the Gateway.

Below are code examples for the most popular Discord Lavalink clients.

### Riffy

```js
const nodes = [
  {
    host: "localhost",
    port: 2333,
    password: "yourpassword",
    secure: false,
    name: "lcluster"
  }
]

const client = new Riffy(discordClient, nodes, {
  send: (payload) => {
    const guild = discordClient.guilds.cache.get(payload.d.guild_id)
    if (guild) guild.shard.send(payload)
  },
  defaultSearchPlatform: "ytmsearch",
  restVersion: "v4"
})
```

### Shoukaku

```js
const nodes = [
  {
    name: 'lcluster',
    url: 'localhost:2333',
    auth: 'yourpassword'
  }
]

const shoukaku = new Shoukaku(new Connectors.DiscordJS(client), nodes)
```

<Note>
Notice that the host and port point to lcluster â€” not to any individual Lavalink node. lcluster handles routing everything behind the scenes. Your bot has no idea there is a whole cluster back there.
</Note>

## Step 5 â€” Play Something

Let's actually push some audio. Show a minimal play example using Riffy:

```js
const player = client.riffy.createConnection({
  guildId: interaction.guild.id,
  voiceChannel: interaction.member.voice.channel.id,
  textChannel: interaction.channel.id,
  deaf: true
})

const resolve = await client.riffy.resolve({
  query: 'lofi hip hop',
  requester: interaction.user
})

const track = resolve.tracks.shift()
player.queue.add(track)
if (!player.playing) player.play()
```
