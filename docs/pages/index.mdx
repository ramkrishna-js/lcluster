Most people outgrow a single Lavalink node eventually. But scaling audio isn't like scaling a website. You can't toss a dumb load balancer in front of it and call it a day, because audio sessions are painfully stateful. If the server dies, the song dies.

So you write custom code in your Discord bot. You maintain lists of nodes. You write heartbeat checkers. You handle moving players between instances. And it's a huge, complicated mess. 

We built **lcluster** so you never have to write that custom code again. It wraps a brilliant Smart Gateway in front of a fleet of nodes that you can manage directly from a beautiful full-screen terminal dashboard. It handles the annoying failovers. It balances the loads. It keeps the music alive.
